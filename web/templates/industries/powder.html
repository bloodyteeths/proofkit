🧮 How we decide PASS/FAIL

Math: Conservative threshold = target + sensor uncertainty; continuous hold window ≥ N minutes; data gaps ≤ G seconds.

📋 Detailed validation criteria
  • Temperature threshold: All readings must exceed (target + sensor_uncertainty). Example: 180°C + 2°C = 182°C minimum.
  • Hold time validation: Continuous mode requires unbroken time above threshold. Cumulative allows brief dips but total time must meet requirement.
  • Ramp rate check: Maximum heating rate calculated using central differences. Excessive rates (>20°C/min) flagged as potential coating defects.
  • Multi-zone logic: For convection ovens with multiple sensors, applies minimum/average/majority consensus as specified.
  • Data quality: Gaps in temperature data exceeding 60 seconds cause automatic failure. Sampling rate must be consistent.
  • Time-to-threshold: Heat-up phase must complete within specified time limit to ensure proper cure kinetics.

❓ Powder coat validation FAQ

🎯 What temperature standards does ProofKit support for powder coating?

ProofKit supports all major powder coating standards including AAMA 2603/2604/2605 for architectural applications, ASTM D523 for gloss retention, and automotive OEM specifications (Ford WSS-M2P175-A1, GM 9984P, etc.). Default settings follow typical 180°C ± 2°C requirements.

🏭 How do I validate multi-zone convection ovens?

Upload CSV files with separate temperature columns for each oven zone (zone_1_temp, zone_2_temp, etc.). ProofKit will analyze all sensors and apply your specified logic (minimum, average, or majority consensus) to determine compliance across all zones.

📈 What's the difference between continuous and cumulative hold time?

Continuous: Part must maintain target temperature for the full duration without dropping below threshold (stricter, recommended for critical applications).

Cumulative: Allows brief temperature dips - total time above threshold must meet requirement (more forgiving for older ovens).

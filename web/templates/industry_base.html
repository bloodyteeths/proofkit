Frequently Asked Questions

Common questions about ProofKit validation for {{ industry_id | title }} applications

What temperature log formats are supported?

ProofKit accepts CSV files with timestamp and temperature columns. Timestamps can be ISO format (2024-01-15T10:00:00Z) or Unix timestamps. Temperature sensors should be labeled clearly (e.g., sensor_1, thermocouple_A). Maximum file size is 10MB with up to 200,000 data points.

How accurate are the validation algorithms?

Our algorithms follow industry-standard validation methods with conservative thresholds. We account for sensor uncertainty, use hysteresis for threshold crossings, and provide deterministic results. The system is designed to be inspector-ready with cryptographic proof of data integrity.

What's included in the proof package?

Each validation generates a PDF report with pass/fail status, temperature charts, specification details, and a QR code for verification. The evidence package includes the original data, processing parameters, and cryptographic hashes for tamper detection.

Can I customize the validation parameters?

Yes, the specification JSON can be modified to match your specific requirements. You can adjust target temperatures, hold times, ramp rates, sensor uncertainty values, and validation logic (continuous vs. cumulative hold time). The interface is pre-filled with industry standards for your convenience.

Is my data stored or shared?

No, ProofKit is designed for privacy and security. Your temperature data is processed in memory and not permanently stored on our servers. The validation is performed locally, and only the results are generated. No authentication or database storage is required.

ğŸ’¡ Understanding ProofKit File Naming Convention

Our example files use descriptive, industry-standard naming that clearly identifies each manufacturing scenario:

  • powder_coat_cure_ - Manufacturing process type (powder coating thermal cure)
  • successful_ or insufficient_hold_time_ - Process outcome or failure mode
  • 180c_10min - Target cure temperature (Celsius) and hold time (minutes)
  • pass or fail - Expected quality control result
  • .csv - Temperature sensor data file (timestamp + temperature columns)
  • .json - Cure specification file (temperature targets, tolerances, sensor rules)

Example: powder_coat_cure_successful_180c_10min_pass.csv represents temperature data from a successful powder coat cure at 180Â°C for 10 minutes that should pass quality validation.

ğŸ”§ Customizing Powder Coat Cure Specifications

You can customize any JSON cure specification to match your manufacturing requirements by editing these critical parameters:

  • target_temp_C: Required cure temperature in Celsius (e.g., 180.0 for standard automotive coatings)
  • hold_time_s: Time at cure temperature in seconds (600 = 10 minutes)
  • sensor_uncertainty_C: PMT sensor measurement uncertainty (default: 2.0Â°C)
  • hysteresis_C: Temperature tolerance band around target (default: 2.0Â°C)
  • max_sample_period_s: Maximum time between temperature readings (60 = 1 minute)
  • allowed_gaps_s: Maximum data gap before failure (120 = 2 minutes)
  • continuous: true for continuous hold requirement, false for cumulative hold
  • sensor_selection.mode: "min_of_set", "max_of_set", or "majority_over_threshold"

Manufacturing Tip: Higher-value parts typically use stricter tolerances (Â±1Â°C) and shorter allowed gaps, while large industrial parts may use cumulative hold logic to accommodate thermal mass variations.
